Debug Northern Shield systematically using evidence from the code.

 COMMON BUG AREAS AND ROOT CAUSES:

 Path blocked on placement:
     grid.findPath() returns null when walls fully enclose spawn or goal.
     game.js checks the return value and reverts the cell — verify this check is in place.

 Enemy teleports on reroute:
     Enemy.setPath() prepends current position: [{ x, y }, ...newPath]
     If this prefix is missing, the enemy jumps to the path start.

 Bullet keeps flying after target dies:
     bullet.update() checks !this.target.alive and sets this.alive = false.
     If enemy.alive is not set to false before bullet processes, it continues.

 Myling not flying straight:
     ENEMY_DEFS.myling.flying must be true.
     spawnEnemy() checks ENEMY_DEFS[type].flying for path type selection.
     rerouteActiveEnemies() must skip enemies where enemy.flying === true.

 Mara EMP not disabling towers:
     ENEMY_DEFS.mara.empRadius must be set.
     game.js update loop must check enemy.type === ENEMY_TYPES.MARA.
     Tower.empDisabled flag must prevent tower.update() from firing.

 Sprite not appearing:
     SPRITES[key] loads asynchronously via Image.onload — may not be ready on first frame.
     drawSpriteFrame() returns false if sprite not loaded — fallback should handle this.
     Check: is the sprite file actually in assets/towers/ and served by Vite?

 Sprite flipping wrongly:
     drawSpriteFrame() uses Math.cos(aimAngle) < 0 to detect left-facing.
     aimAngle must be calculated correctly: Math.atan2(dy, dx) from tower to target.

 Canvas state bleeding between draws:
     shadowBlur not reset → all subsequent draws have unintended glow.
     setLineDash not reset → subsequent lines are dashed.
     textAlign not restored → text alignment wrong after HUD draw.
     Use ctx.save() / ctx.restore() or explicitly reset after each use.

 HUD layout (for reference):
     Canvas fills window. Grid starts at (GRID_LEFT, GRID_TOP) = (188, 56).
     LEFT_DOCK_W=172 (left), RIGHT_PANEL_W=188 (right), GRID_TOP=56 (War Room top bar).
     Grid pixel size: 48*14 = 672px wide, 30*14 = 420px tall.

 DEBUGGING PROCESS:
 1. Identify the symptom precisely
 2. Trace which file/function owns the broken behavior
 3. Propose the smallest safe fix
 4. State side effects of the fix
 5. Do not rewrite unrelated systems to fix a localized bug
