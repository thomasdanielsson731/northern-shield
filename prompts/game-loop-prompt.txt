Keep Northern Shield's game loop deterministic and easy to reason about.

 CURRENT LOOP (src/core/game.js):
     function gameLoop() {
         update()       // all logic
         draw()           // all rendering
         requestAnimationFrame(gameLoop)
     }

 UPDATE ORDER (order matters — do not change without reason):
     1. Wave state machine (countdown / active / break) + spawn queue tick
     2. towers.forEach → tower.update(enemies, bullets)
               pushes new Bullet into bullets[], returns 0 (kills handled by bullet)
     3. bullets reverse iterate → bullet.update()
               moves bullet, on hit: sets enemy.alive=false, returns enemy.reward (>0 = kill)
     4. enemies reverse iterate → enemy.update()
               moves enemy, checks reached/alive, Banshee EMP radius check
     5. particles reverse iterate → move, decay life, splice when dead
     6. Screen shake: shakeX, shakeY decay toward 0 each frame

 WAVE STATE MACHINE:
     'countdown': COUNTDOWN_FRAMES=150 frames (~2.5s) — shows countdown timer
     'active':     spawn queue ticks down, enemies spawn from queue
                         wave ends when spawn queue empty AND enemies[] is empty
     'break':       BREAK_FRAMES=250 frames (~4s) — rest phase between waves

 DRAW ORDER:
     drawBackground() → grid.draw() → drawPath() → towers → bullets → enemies
     → particles → drawTopBar() → drawLeftSidebar() → drawRightPanel() → drawBottomBuildBar()
     Enemies drawn after towers so they appear on top.
     HUD elements always drawn last (on top of everything).
     Grid translated by (GRID_LEFT+shakeX, GRID_TOP+shakeY) in draw().

 WHY REVERSE ITERATION:
     Bullets, enemies, and particles are spliced during iteration.
     Reverse iteration ensures splice(i, 1) doesn't skip the next element.
     Never iterate forward and splice in the same loop.

 SIDE EFFECTS TO TRACK:
     - tower.update() mutates bullets[] (push)
     - bullet.update() mutates enemy.alive, enemy.hp (direct reference)
     - Banshee EMP mutates tower.empDisabled flag on nearby towers
     - game.js reverse loops splice enemies[] and bullets[]
     - Wall placement calls grid.findPath() synchronously — safe at 50x30 grid size

 Avoid hidden side effects during iteration.
