Design gameplay systems for Northern Shield that are easy to extend, balance, and debug.

 CURRENTLY IMPLEMENTED:
 - Grid (50x30, CELL_SIZE=14, BFS pathfinding) — src/grid/grid.js
 - Wave system: 'countdown' → 'active' → 'break' state machine — src/core/game.js
     COUNTDOWN_FRAMES=150, BREAK_FRAMES=250, wave-driven spawn queues
 - 6 tower types: Shield Wall, Berserker, Valkyrie, Archer, Catapult, Blondie
 - 4 enemy types: Graveborn, Wisp (flying), Golem (tank), Banshee (EMP)
 - Tower targeting: highest pathIndex in range (furthest along path) — src/entities/tower.js
 - Homing bullets with gold trail — src/entities/bullet.js
 - Wall placement with live path validation (BFS before commit) — src/core/game.js
 - Active enemy rerouting when grid changes — src/core/game.js
 - Flying enemies (Wisp) follow straight line, bypass walls entirely
 - Banshee EMP: disables nearby towers within radius when passing — src/core/game.js
 - Shield Wall slow aura: slows enemies in range (slowFactor<1)
 - Blondie stun: sets enemy.slowFactor=0 on hit for N frames
 - Tower upgrades: max MAX_LEVEL=10, scales damage/range/fireRate
 - High scores: localStorage, top 10 entries persist across sessions
 - Screen shake: shakeX/shakeY offset applied in draw()
 - Particle system: plain particles[] array, {x,y,vx,vy,life,maxLife,color}
 - Sprite animations: 4-frame strips (IDLE/WALK/ATTACK/DEATH), drawSpriteFrame() helper

 DATA-DRIVEN PATTERNS IN USE:
 - ENEMY_DEFS drives all enemy stats + visual config (enemy.js)
 - TOWER_DEFS drives all tower stats + button labels + hotkeys (tower.js)
 - BUILD_ITEMS in game.js auto-derives from TOWER_DEFS — adding a tower type
     only requires an entry in TOWER_DEFS and a draw method
 - Enemy.reward is returned by bullet.update() on kill — gold and kill tracking
     happen in game.js from this single return value

 DESIGN RULES:
 - New enemy/tower types should only require adding to ENEMY_DEFS / TOWER_DEFS
 - Avoid tightly coupled gameplay logic
 - Keep systems easy to tune (all numbers in DEF objects, never hardcoded inline)
 - Simple to debug — no hidden state transformations
