Optimize Northern Shield for a stable 60fps frame rate.

 CURRENT RENDERING LOAD PER FRAME:
 - drawBackground(): 1 linear gradient + 2 radial gradients (recreated each frame — acceptable)
 - grid.draw(): up to 1500 cells, EMPTY cells skipped (fast)
 - drawPath(): one polyline over BFS path length (~50-150 nodes typically)
 - Enemies: typically 5-20 active, each: 1-3 arc() calls + HP bar
 - Towers: typically 5-20, each: range ring + sprite drawImage() or canvas shapes
 - Bullets: typically 5-30, each: trail polyline + glow circle
 - Particles: up to ~20 at peak, each: small fillRect() or arc()
 - HUD: drawTopBar, drawLeftSidebar (5 tabs), drawRightPanel, drawBottomBuildBar

 SPRITE RENDERING:
 - drawImage() is fast — no performance concerns at current tower counts
 - SPRITES[key] check before draw is free (simple property lookup)
 - drawSpriteFrame() uses ctx.save/restore + translate/scale — acceptable per tower

 KNOWN EXPENSIVE OPERATIONS:
 - ctx.shadowBlur — non-zero shadowBlur affects ALL subsequent draw calls.
     Always reset to 0 immediately after each glow effect.
 - createRadialGradient in drawBackground() — recreated every frame.
     Acceptable at current scale. Do not cache unless FPS drops below 45.
 - BFS findPath() — called only on wall placement/removal, never per-frame. Correct.
 - ctx.setLineDash() — must be reset with setLineDash([]) after each use.

 HOT PATH RULES:
 - Avoid array.map/filter/reduce inside update() — use plain for loops
 - Avoid object allocation inside the game loop (no `new` in update/draw hot paths)
 - Do not sort enemies array every frame — tower.update() uses a single linear scan
 - Enemy.update() and Bullet.update() are called 60x/second — keep them minimal

 MEASURE FIRST:
 - Use browser DevTools Performance tab before optimizing anything
 - Do not optimize drawBackground() unless measured FPS < 45
 - The BFS does not need optimization for a 50x30 grid
